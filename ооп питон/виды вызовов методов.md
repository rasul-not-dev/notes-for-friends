
```
class Vector:
    def __init__(self, x, y):
        self.x = x
        self.y = y

    # Обычный метод
    def get_coords(self):
        return self.x, self.y

# Создаем объект (экземпляр) класса
v = Vector(1, 2)
```

res = v.get_coords() print(res) # Выведет: (1, 2) --- первый способ

res = Vector.get_coords(v) print(res) # Выведет: (1, 2) --- второй способ


### 2. Методы класса (`@classmethod`)

Используются, когда логика метода связана не с отдельным объектом, а со всем классом в целом (например, общие проверки или альтернативные конструкторы).

- **Декоратор:** `@classmethod`
    
- **Особенность:** Первым аргументом автоматически принимает `cls` (ссылку на сам класс, а не на его объект).
    
- **Доступ:** Имеет доступ только к атрибутам класса (например, `cls.MIN_COORD`), но «не видит» свойства конкретных объектов.
    
- **Вызов:** Вызывается напрямую через класс: `Vector.validate(5)`.
    
- **💡 Совет по Django из видео:** Внутри других методов класса (например, в `__init__`) вызывать метод класса лучше универсально: через `self.validate(x)` или `cls.validate(x)`. Это избавляет от жесткой привязки к имени класса `Vector`.
    

### 3. Статические методы (`@staticmethod`)

Это обычные изолированные функции, которые логически связаны с тематикой класса, поэтому их поместили внутрь него для удобства организации кода.

- **Декоратор:** `@staticmethod`
    
- **Особенность:** Не принимают никаких скрытых аргументов (`ни self`, ни `cls`). В скобках будут только те параметры, которые ты указал сам.
    
- **Доступ:** Полная изоляция. Метод не должен зависеть ни от класса, ни от его объектов. Он просто берет аргументы из скобок, считает и возвращает результат.
    
- **Пример из видео:** Метод вычисления квадратичной нормы по формуле $x^2 + y^2$. Ему не важно, откуда пришли $x$ и $y$.
    
- **Вызов:** `Vector.get_norm(5, 6)`.
