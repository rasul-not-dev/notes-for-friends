a = Point()
b = Point()

**Как работают пространства имен (Локальные vs Общие атрибуты)**
Это один из самых важных моментов лекции, который часто путают:

Изначально у созданных объектов a и b внутри ничего нет. Их личный локальный словарь свойств a.__dict__ пуст ({}).

Когда мы запрашиваем a.color, Python не находит это свойство внутри a, поэтому он идет во внешнее пространство имен — в класс Point — и берет значение оттуда.

Если изменить атрибут в классе (Point.circle = 1), то значение автоматически изменится у всех объектов (a.circle и b.circle тоже станут 1), так как они ссылаются на общие данные класса.

Магия оператора присваивания:
Если мы напишем:

a.color = 'green'

Python не меняет значение в самом классе Point. Вместо этого он создает новое локальное свойство color лично внутри объекта a.

Теперь a.__dict__ вернет {'color': 'green'}.

Объект b при этом по-прежнему будет брать общую строку 'red' из класса Point.




Чтобы превратить объекты в реальные независимые точки, мы динамически задаем им персональные координаты `x` и `y`, которые будут храниться только в их личных словарях `__dict__`:

```
a = Point()
a.x = 1
a.y = 2

b = Point()
b.x = 10
b.y = 20
```

_Результат:_ `a` и `b` имеют общие переменные `color` и `circle` (из класса), но абсолютно уникальные и независимые координаты `x` и `y`.
