### 1. Наследование (Inheritance)

**Что это:** Способность одного класса (дочернего) перенять все свойства и методы другого класса (родительского). **Зачем нужно:** Чтобы не копировать один и тот же код сто раз.

- **Родительский класс (Базовый):** Тот, у кого берут основу.
    
- **Дочерний класс:** Тот, кто берёт основу и может добавлять своё.
    

```
# Родительский класс
class Animal:
    def __init__(self, name):
        self.name = name
    
    def eat(self):
        print(f"{self.name} кушает.")

# Дочерний класс (наследует Animal)
class Cat(Animal):
    def meow(self):
        print("Мяу!")
```

_У класса `Cat` автоматически есть и имя, и метод `eat`, хотя мы их внутри него не писали._

### 2. Переопределение методов (Method Overriding)

**Что это:** Когда дочерний класс переписывает метод родителя, чтобы изменить его поведение.

```
class Cow(Animal):
    # Переопределяем метод eat специально для коровы
    def eat(self):
        print(f"{self.name} жует траву.")
```

### 3. Функция `super()`

**Что это:** Команда, которая вызывает метод родительского класса из дочернего. **Зачем нужна:** Чтобы не стирать код родителя при переопределении, а **дополнять** его.

- **Без `super()`:** Дочерний метод полностью стирает код родительского метода.
    
- **С `super()`:** Сначала выполняется стандартный код родителя, а затем добавляется новый код дочернего класса.
    

#### Пример в коде:

```
class ProCat(Animal):
    def __init__(self, name, lives):
        # super() подтягивает создание имени из родительского класса Animal
        super().__init__(name) 
        
        # Добавляем уникальное свойство именно для ProCat
        self.lives = lives 
``` 

### 🚀 Зачем это нужно в Django?

В Django ты **всегда** наследуешься от готовых классов. `super()` чаще всего нужен при сохранении данных в базу (`save()`), чтобы добавить своё действие, но не сломать стандартную логику Django:

```
class Article(models.Model): # Наследование от Django
    title = models.CharField(max_length=200)

    # Переопределяем встроенный метод сохранения
    def save(self, *args, **kwargs):
        # 1. Делаем свое действие (например, приводим заголовок к нижнему регистру)
        self.title = self.title.lower() 
        
        # 2. Вызываем super(), чтобы Django записал всё в базу данных своим сложным кодом
        super().save(*args, **kwargs) 
```
