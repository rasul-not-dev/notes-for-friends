#### 1. Наследование (Inheritance)

_В курсе это видео #15–#20._

- **Что это:** Способность класса перенимать свойства и методы родительского класса.
    
- **Зачем в Django:** Это фундамент Django. Всё, что ты пишешь в Django, наследуется от встроенных классов:
    
    - Модели базы данных: `class Post(models.Model):` — твой класс поста наследует всю логику работы с БД от `models.Model`.
        
    - Классовые представления: `class HomeView(ListView):` — представление для списка статей наследует логику отображения от `ListView`.
        

#### 2. Функция `super()` и переопределение методов

_В курсе это видео #17._

- **Что это:** Позволяет изменить поведение метода родительского класса, но при этом вызвать оригинальный код родителя, чтобы ничего не сломать.
    
- **Зачем в Django:** Очень часто нужно переопределить сохранение формы или модели, например, автоматически сгенерировать URL (slug) перед сохранением статьи:
    
    Python
    
    ```
    def save(self, *args, **kwargs):
        if not self.slug:
            self.slug = generate_slug(self.title)
        super().save(*args, **kwargs) # Вызываем стандартное сохранение Django
    ```
    

#### 3. Свойства `property` и декоратор `@property`

_Это как раз видео #9, информацию по которому мы сейчас подняли._

- **Что это:** Позволяет вызывать метод класса так, будто это обычный атрибут (без круглых скобок `()`), делая код чище.
    
- **Зачем в Django:** 1. **В моделях:** Если у тебя в модели юзера есть `first_name` и `last_name`, ты можешь сделать виртуальное поле `full_name`:
    
    Python
    
    ```
    @property
    def full_name(self):
        return f"{self.first_name} {self.last_name}"
    ```
    
    В шаблоне HTML ты сможешь написать просто `{{ user.full_name }}` без скобок. 2. **В корзине покупок:** Считать общую сумму товаров: `cart.total_price`.
    

#### 4. Магический метод `__str__`

_В курсе это видео #11._

- **Что это:** Определяет, как объект класса будет выглядеть, если его принудительно перевести в строку (например, через `print()`).
    
- **Зачем в Django:** Без этого метода в админ-панели Django вместо названий твоих статей (например, «Как учить Django») будет отображаться нечитаемый текст вида `<Post object (1)>`.
    
    Python
    
    ```
    class Post(models.Model):
        title = models.CharField(max_length=200)
    
        def __str__(self):
            return self.title # Теперь в админке будет красивое название!
    ```
    

#### 5. Декораторы `@classmethod` и `@staticmethod`

_В курсе это видео #6._

- **Что это:** `@classmethod` принимает первым аргументом сам класс (`cls`), а не его экземпляр (`self`).
    
- **Зачем в Django:** Редко, но используется для кастомных методов в менеджерах моделей (например, чтобы сделать метод `Post.objects.get_published()`).
