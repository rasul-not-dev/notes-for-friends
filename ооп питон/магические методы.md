#### Таблица методов и их правильного вызова

| **Метод**              | **Когда вызывается?**                             | **Главная опасность**                      | **Как делать правильно?**                          | **Зачем это нужно в реальном коде?**                                                                                                                   |
| ---------------------- | ------------------------------------------------- | ------------------------------------------ | -------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------ |
| **`__getattribute__`** | При **любом** чтении любого атрибута.             | Бесконечная рекурсия при `self.name`       | Использовать `super().__getattribute__(item)`      | **Логирование и аудит:** Записать в базу, кто и когда читал секретные данные. **Проверка прав:** Запретить читать данные, если юзер не админ.          |
| **`__getattr__`**      | Только если атрибут **не нашли**.                 | Безопасен, рекурсии обычно нет.            | Возвращать дефолтное значение или вызывать ошибку. | **Динамические свойства:** Например, превратить JSON из API в объект, где ключи становятся атрибутами (`user.age`), даже если они не прописаны в коде. |
| **`__setattr__`**      | При **любой** записи или изменении атрибута.      | Бесконечная рекурсия при `self.name = val` | Использовать `super().__setattr__(key, val)`       | **Валидация данных:** Автоматически проверять, что в `email` пришла строка с собачкой `@`, а в `age` — положительное число.                            |
| **`__delattr__`**      | При **любом** удалении атрибута (`del obj.attr`). | Бесконечная рекурсия при `del self.name`   | Использовать `super().__delattr__(item)`           | **Защита данных:** Запретить удалять критически важные поля (например, системные ID или статус пользователя).                                          |
