## Урок 7: Слияние веток (Git Merge) — Базовый сценарий

Когда ты полностью закончил работу над фичей в отдельной ветке (например, `add-dark-mode`) и протестировал её, пришло время объединить этот код с основной веткой `master`. Этот процесс называется **слиянием (Merge)**.

### 🗺️ Главный алгоритм слияния (Всего 2 шага)

Запомни железное правило: **ты всегда должен находиться в той ветке, В КОТОРУЮ хочешь влить изменения.**

Если ты хочешь перенести код из ветки `feature` в ветку `master`:

1. **Переключаешься в главную ветку:**
    
    Bash
    
    ```
    git checkout master
    ```
    
2. **Вызываешь команду слияния, указывая имя ветки-донора:**
    
    Bash
    
    ```
    git merge feature
    ```
    

### ⚡ Тип слияния №1: Fast-Forward (Перемотка вперед)

В этом уроке подробно разбирается самый простой и чистый сценарий слияния — **Fast-Forward**.

- **Когда это происходит:** Когда ты создал ветку `feature` от ветки `master`, ушел в нее работать и делал там коммиты, а в самой ветке `master` в это время **ничего не менялось** (она стояла на месте).
    
- **Как это делает Git:** Гит понимает, что история прямая, как стрела. Он не придумывает ничего сложного, а просто берет указатель `master` и мгновенно перемещает (перематывает) его вперед — на самый последний коммит ветки `feature`.
    
- **Результат:** Файлы в твоем `master` мгновенно обновляются, история проекта остается идеально прямой линией, а ветку `feature` теперь можно спокойно удалять (`git branch -d feature`), так как `master` теперь содержит абсолютно весь её код.
    

### 💡 Вывод для Обсидиана

Слияние по типу _Fast-Forward_ — это идеальный сценарий для соло-разработчика. Ты просто переключаешься на `master`, пишешь `git merge <имя_ветки>` — и Гит за долю секунды накатывает твои новые фичи на старый рабочий код.

Логика понятна? Готов перейти к **8-му уроку**? Там этот процесс усложняется, и мы разберем, что делать, если история пошла по двум разным дорогам одновременно.




**ШПАРГАЛКА**

Представь, что ты написал код для новой фичи в ветке `super-code`. Всё работает, и ты хочешь закинуть этот код в главную ветку `master`.

Твои действия:

1. Пишешь в консоли `git checkout master` (переходишь в главную ветку).
    
2. Пишешь `git merge super-code` (говоришь Гиту: "притяни сюда всё из super-code").
    

Если пока ты кодил в `super-code`, саму ветку `master` ты не трогал, то сработает **Fast-Forward**. Гит просто переставит ярлык `master` на твой самый свежий коммит. Всё, код объединился за долю секунды!

После этого ветка `super-code` больше не нужна, её можно стереть командой `git branch -d super-code`, чтобы под ногами не путалась.
