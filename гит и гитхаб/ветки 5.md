### Как избежать конфликтов (Твой золотой стандарт)

Просто **вообще не трогай ветку `master`**, пока работаешь в другой ветке. Делай коммиты только во `feature`, а потом вливай её в замерший `master`. Гит всегда будет делать авто-слияние (_Fast-Forward_), и конфликтов не случится никогда.

### Единственный случай, когда ты поймаешь конфликт

Если ты сделал коммиты во `feature`, но перед слиянием зашел в `master` и **в тех же самых строках** кода тоже сделал коммит (например, срочно правил баг).

### Что делать, если конфликт всё-таки случился:

1. Гит остановит мердж и покажет в коде маркеры:
    
    HTML
    
```
    <<<<<<< HEAD
    Код из ветки master
    =======
    Код из ветки feature
    >>>>>>> feature
```
    
2. Руками сотри маркеры (`<<<<<<<`, `=======`, `>>>>>>>`) и оставь только тот HTML-код, который правильный.
    
3. Сохрани файл и заверши мердж финальным коммитом:
    
    Bash
    
```
    git add .
    git commit -m "Разрешил конфликт"
```
