## 🗺️ Карта взаимодействия

```
  [ GitHub (Облако) ]  <--- Ссылка (URL) берется на сайте репозитория
       |         ^
 1.    |         | 3.
 Clone |         | Push
       v         |
  [ Компьютер (Локально) ] <--- Хэш (ID коммита) берется через 'git log'
       |         ^
       +----+----+
         2. | Pull
```

## 🛠️ Справочник команд и действий

### 1. `git clone <ссылка>` — Скачать проект с нуля

- **Где брать ссылку:** На сайте GitHub в нужном репозитории нажмите зеленую кнопку **Code** -> вкладка **HTTPS** -> скопируйте URL.
    
- **Куда вводить:** В консоли на ПК (в папке, где должен появиться проект). Выполняется **1 раз** в начале.
    
- **Что делать дальше:** Сразу перейти в скачанную папку через `cd <имя_папки>`.
    

### 2. `git remote add origin <ссылка>` — Связать свой локальный проект с GitHub

- **Когда нужно:** Когда ты сначала создал проект на ПК через `git init`, а потом хочешь загрузить его на созданный пустой репозиторий на GitHub.
    
- **Где брать ссылку:** GitHub сам покажет её на главной странице только что созданного пустого репозитория.
    

### 3. `git push -u origin <имя_ветки>` — Отправить код в облако

- **Что делает:** Выгружает твои локальные коммиты на GitHub.
    
- **Флаг `-u`:** Нужен только при самом первом пуше в ветку. Дальше можно писать просто `git push`.
    

### 4. `git pull` — Забрать обновления из облака

- **Когда нужно:** Если твой напарник закинул новый код на GitHub или ты сам правил файлы через сайт. Команда скачивает изменения и вливает их в твой локальный код.


#### 🏃‍♂️ Короткий чек-лист: Обычный рабочий цикл

1. `git pull` — проверил, нет ли новинок из облака.
    
2. Поработал, написал код.
    
3. `git add .` — добавил в индекс.
    
4. `git commit -m "что сделал"` — зафиксировал локально.
    
5. `git push` — отправил результат на GitHub.
