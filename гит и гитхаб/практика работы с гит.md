
Сначала очень простое объяснение «на пальцах», чтобы закрепить связь между ними:

- **Git** — это твоя личная локальная «система бэкапов и контроля версий». Она работает прямо на твоем компьютере (в консоли). Git следит за файлами в твоей папке, запоминает изменения, позволяет создавать ветки (альтернативные реальности проекта) и возвращаться в прошлое. Ему вообще не нужен интернет.

- **GitHub** — это удаленный сайт (облако) в интернете. Туда ты отправляешь копию своего локального проекта (который ведет Git), чтобы твои файлы не пропали, если сгорит компьютер, или чтобы показать их другим разработчикам.


### Шаг 1. Готовим наш «проект»

Давай создадим на твоем компьютере (например, на Рабочем столе или в любой удобной папке) новую папку. Назовем её **`my-first-space-odyssey`** (Моя первая космическая одиссея). Представим, что мы пишем текстовую игру про полет на Марс.

Внутри этой папки создай вручную два обычных текстовых файла. Вот тебе их содержимое, скопируй и вставь:

**Файл №1: `crew.txt`** (Список экипажа)

Plaintext

```
Капитан: [Твое Имя]
Бортинженер: Робот БЕН
Связист: Алиса
```

**Файл №2: `ship.txt`** (Состояние корабля)

Plaintext

```
Название корабля: "Восток-2026"
Топливо: 100%
Системы жизнеобеспечения: Стабильно
```

Теперь у тебя есть папка `my-first-space-odyssey`, в которой лежат два файла. Пока что это просто файлы на Windows. Git про них ничего не знает.

### Шаг 2. Оживляем Git (Работаем на компьютере)

Открой свою консоль (Git Bash или командную строку Windows) и перейди в созданную папку.

_(Если пользуешься Git Bash, можно просто кликнуть правой кнопкой мыши по пустой области внутри папки `my-first-space-odyssey` и выбрать "Git Bash Here")_.

Давай начнем вводить команды. Я буду подробно объяснять каждую.

#### 1. Инициализация репозитория

Bash

```
git init
```

**Что происходит:** Ты говоришь Git: «Всё, парень, с этого момента ты следишь за этой папкой». Git создает внутри скрытую папку `.git` (ты её не увидишь, если не включено отображение скрытых файлов). Теперь эта папка — **локальный репозиторий**.

#### 2. Проверяем статус

Bash

```
git status
```

**Что увидишь:** Git подсветит файлы `crew.txt` и `ship.txt` красным цветом. Он скажет, что эти файлы **Untracked** (неотслеживаемые). То есть он их видит, но в свою память еще не записал.

#### 3. Подготавливаем файлы к сохранению

Bash

```
git add .
```

_(Точка в конце означает «добавить ВСЕ файлы в этой папке»)_.

**Что происходит:** Ты переносишь файлы на «сцену» (Staging area). Это промежуточный этап. Если ты снова введешь `git status`, файлы станут зелеными. Git готов их запомнить.

#### 4. Делаем первое сохранение (Коммит)

Bash

```
git commit -m "Initial commit: crew and ship added"
```

Флаг `-m` означает "message" (комментарий). В кавычках мы пишем понятным языком, что именно мы сделали.

**Что происходит:** Git сделал снимок (бэкап) твоих файлов в их текущем состоянии. Это точка сохранения, к которой ты всегда сможешь вернуться. Поздравляю, у тебя создалась главная ветка (обычно она называется `master` или `main`).

### Шаг 3. Эксперименты с ветками (Альтернативная реальность)

Представь, что ты хочешь добавить в игру новую фичу — "Гипердвигатель", но боишься сломать то, что уже работает стабильно. Для этого мы создадим **ветку** (branch). Ветка — это копия твоего проекта, которая развивается отдельно.

#### 1. Создаем новую ветку и переходим в неё

Bash

```
git checkout -b feature-hyperdrive
```

_(Флаг `-b` создает новую ветку, а `checkout` переключает тебя на неё)_.

**Что происходит:** Теперь ты находишься в ветке `feature-hyperdrive`. Всё, что ты будешь делать здесь, никак не повлияет на главную ветку.

#### 2. Экспериментируем

Открой файл `ship.txt` через обычный Блокнот и допиши туда четвертую строчку:

Plaintext

```
Название корабля: "Восток-2026"
Топливо: 100%
Системы жизнеобеспечения: Стабильно
Двигатель: Экспериментальный Гипердвигатель (Установлен)
```

Сохрани файл.

#### 3. Сохраняем изменения в новой ветке

В консоли проделываем уже знакомые шаги:

Bash

```
git add ship.txt
git commit -m "Added hyperdrive to the ship"
```

Теперь в ветке `feature-hyperdrive` у корабля есть гипердвигатель.

#### 4. Магия Git: Перемещение во времени

Давай вернемся в главную ветку. Введи команду:

Bash

```
git checkout main
```

_(Если Git ругается, что ветки `main` нет, попробуй `git checkout master` — имя зависит от настроек твоей консоли)_.

**А теперь быстро открой файл `ship.txt` в Блокноте!**

Четвертая строчка про гипердвигатель исчезла! Как будто её и не было. Git мгновенно перестроил файлы на твоем компьютере под состояние главной ветки.

Если ты снова введешь `git checkout feature-hyperdrive`, строчка вернется. Правда круто?

#### 5. Слияние веток (Merge)

Эксперимент удался, гипердвигатель работает отлично, давай добавим его в основной проект.

Перейди в главную ветку (если ты еще не в ней):

Bash

```
git checkout main
```

И дай команду на слияние:

Bash

```
git merge feature-hyperdrive
```

**Что произошло:** Git взял изменения из ветки с двигателем и перенес их в главную ветку. Теперь, если ты откроешь `ship.txt`, строчка про гипердвигатель будет там навсегда. Ветку `feature-hyperdrive` теперь можно даже удалить за ненадобностью, но пока оставим.

### Шаг 4. Подключаем GitHub (Выходим в облако)

Пока что всё это происходило локально у тебя на жестком диске. Теперь свяжем это с GitHub.

1. Зайди на сайт **github.com** под своим аккаунтом.
    
2. В верхнем углу нажми на плюсик `+` -> **New repository** (Новый репозиторий).
    
3. В поле **Repository name** введи точно такое же имя: `my-first-space-odyssey`.
    
4. Больше ничего не трогай (не ставь галочки напротив README, .gitignore — они у нас уже как бы есть). Нажми внизу зеленую кнопку **Create repository**.
    

Перед тобой откроется страница с подсказками. Нам нужен блок, который называется **"…or push an existing repository from the command line"** (или отправить существующий репозиторий из командной строки).

Там будет три строчки кода. Скопируй их и по очереди введи в свою консоль:

Bash

```
# 1. Эта команда связывает твою локальную папку с репозиторием на сайте. 
# (Ссылка у тебя будет своя, с твоим никнеймом на Гитхабе)
git remote add origin https://github.com/твой-никнейм/my-first-space-odyssey.git

# 2. Эта команда гарантирует, что твоя главная ветка называется именно main
git branch -M main

# 3. А эта команда отправляет (push) твои файлы и всю историю коммитов в облако GitHub
git push -u origin main
```

_(При первом push Windows может попросить тебя авторизоваться в браузере или ввести токен от GitHub, чтобы подтвердить, что это твой аккаунт — просто следуй инструкциям на экране)_.

**Готово!** Обнови страницу репозитория на GitHub в браузере. Ты увидишь свои файлы `crew.txt` и `ship.txt`. Если ты кликнешь по кнопке коммитов, ты увидишь всю историю: кто, когда и какие строчки добавлял.

### Твое домашнее задание для закрепления:

1. Прямо на компьютере в папке создай третий файл `log.txt` (бортовой журнал). Напиши туда: _"День 1. Полет нормальный"_.
    
2. Сделай `git add log.txt`
    
3. Сделай коммит: `git commit -m "Added captain log"`
    
4. Отправь изменения на GitHub с помощью короткой команды: `git push`
    

Попробуй сделать эти шаги самостоятельно. Если на каком-то этапе консоль выдаст ошибку или что-то пойдет не так — скопируй текст ошибки сюда, и мы детально разберем, что произошло!
