
 Пока ты работаешь локально, созданные ветки (`feature-1`, `test-styles`) видишь только ты на своем компьютере. На GitHub отправляется только то, что ты пушишь явно. В этом уроке разбирается, как синхронизировать ветки с облаком и как наводить порядок.

### 📤 1. Как отправить локальную ветку на GitHub (Push)

Если ты создал новую ветку `add-dark-mode`, поработал в ней, сделал коммиты и хочешь сохранить эту ветку в облаке (для бэкапа или чтобы продолжить работу с другого компа), обычный `git push` не сработает. Гит скажет, что в облаке еще нет такой ветки.

**Правильная команда для первого пуша новой ветки:**

> `git push -u origin <имя_ветки>`

- _Пример:_ `git push -u origin add-dark-mode`
    
- Флаг `-u` (upstream) связывает твою локальную ветку с созданной веткой на GitHub. Все последующие разы, находясь в этой ветке, ты сможешь писать просто короткое: `git push`.
    

### 🔄 2. Как скачать ветку с GitHub, которой нет на компе (Fetch & Checkout)

Представь ситуацию: ты запушил ветку с рабочего ПК, пришел домой, сел за ноутбук. На ноуте этой ветки пока нет.

1. Сначала нужно обновить информацию о том, что вообще изменилось на GitHub:
    
    > `git fetch` _(Гит проверит облако и узнает, что там появилась новая ветка)._
    
2. Теперь ты можешь просто переключиться на нее:
    
    > `git checkout <имя_ветки>` _(Гит поймет, что в облаке она есть, создаст такую же у тебя локально и свяжет их)._
    

### 🧹 3. Удаление веток (Наводим порядок)

Когда задача выполнена (например, фича полностью готова и в следующих уроках мы сольем её в `master`), ветка становится ненужным хламом. Ее нужно удалить, чтобы дерево проекта оставалось чистым.

**Удаление локально (на твоем компьютере):**

> `git branch -d <имя_ветки>`

- _Нюанс:_ Если ты попытаешься удалить ветку, код из которой ты еще никуда не слил, Гит выдаст ошибку, защищая тебя от потери данных.
    
- Если ты точно уверен, что хочешь удалить ветку со всем багнутым кодом навсегда, используй жесткое удаление с заглавной буквой: `git branch -D <имя_ветки>`.
    

**Удаление удаленно (на GitHub):** Удаление ветки на компьютере не удаляет её из облака автоматичеки. Чтобы стереть её и на GitHub, пишется команда:

> `git push origin --delete <имя_ветки>`

### 💡 Вывод для Обсидиана

GitHub может хранить сколько угодно твоих параллельных веток. Для соло-разработчика пуш отдельных веток — это крутой способ делать «точки сохранения» сложных фич в облаке, не засоряя основную стабильную ветку `master/main`. Сделал задачу — запушил — проверил — удалил лишнее.





**ШПАРГАЛКА**

- `git push -u origin <имя_ветки>` — отправить новую ветку на GitHub и связать её с локальной (флаг `-u` нужен только в первый раз, дальше можно просто `git push`).
    
- `git fetch` — обновить данные с GitHub (чтобы локальный Git узнал о ветках, созданных в облаке или на другом ПК).
    
- `git checkout <имя_ветки>` — скачать и переключиться на ветку, которая уже есть на GitHub, но еще не появилась на компе.
    
- `git branch -d <имя_ветки>` — безопасно удалить ветку на компьютере (если код не слит в main, Git выдаст ошибку).
    
- `git branch -D <имя_ветки>` — принудительно удалить ветку локально вместе со всем кодом (без предупреждений).
    
- `git push origin --delete <имя_ветки>` — удалить ветку в облаке на GitHub.
    

**Суть на пальцах:** Пока ты не сделаешь пуш, твои побочные ветки никто на GitHub не увидит. Локальное удаление ветки и удаление на GitHub — это два разных действия, чистить нужно и там, и там. Пушить ветки в облако удобно для бэкапа и работы с разных компьютеров.
