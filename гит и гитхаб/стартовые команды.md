## 🧭 Навигация (Как ходить по папкам)

Когда ты открываешь консоль, ты находишься в «домашней» папке. Чтобы перемещаться по компьютеру, используй эти команды:

- **`pwd`** (Print Working Directory — _показать рабочую папку_)
    
    - **Что делает:** Выводит на экран полный путь к папке, в которой ты сейчас находишься. Полезно, если ты «заблудился».
        
- **`ls`** (List — _список_)
    
    - **Что делает:** Показывает все файлы и папки внутри текущей директории.
        
- **`ls -a`** (List All — _показать всё_)
    
    - **Что делает:** Показывает вообще все файлы, включая скрытые. Именно с этой командой ты сможешь увидеть скрытую папку `.git`, которую создает Гит для слежки за проектом.
        
- **`cd [имя_папки]`** (Change Directory — _сменить папку_)
    
    - **Что делает:** Заходит внутрь указанной папки. Например: `cd Desktop` (перейти на рабочий стол).
        
- **`cd ..`** (Две точки)
    
    - **Что делает:** Шаг назад. Выходит из текущей папки на один уровень вверх.
        

## 🛠️ Работа с файлами и папками (Управление)

Эти команды заменяют привычные клики мышкой (создать папку, создать файл):

- **`mkdir [имя_папки]`** (Make Directory — _создать папку_)
    
    - **Что делает:** Создает новую пустую папку в том месте, где ты находишься. Например: `mkdir my-project`.
        
- **`touch [имя_файла.расширение]`**
    
    - **Что делает:** Создает пустой файл. Например: `touch index.html` или `touch script.js`.
        
- **`rm [имя_файла]`** (Remove — _удалить_)
    
    - **Что делает:** Навсегда удаляет файл. **Внимание:** файлы удаляются мимо корзины, вернуть их будет сложно!
        
- **`rm -rf [имя_папки]`**
    
    - **Что делает:** Жестко и безвозвратно удаляет папку со всем её содержимым. Используй очень осторожно.
        

## 🐙 Первые команды Git (Твой фундамент из 1 видео)

А это те самые команды из первого урока, которые ты будешь вводить тысячи раз:

- **`git init`** (Initialize — _инициализировать_)
    
    - **Что делает:** Включает Git в текущей папке. С этого момента Git начинает следить за всеми изменениями здесь.
        
- **`git status`** (_статус_)
    
    - **Что делает:** Показывает текущее состояние твоего проекта. Какие файлы изменились (красные), какие готовы к сохранению (зеленые), а какие Git вообще еще не видит.
        
- **`git add [имя_файла]`** или **`git add .`** (Точка — значит ВСЁ)
    
    - **Что делает:** Переносит файлы в «черновик» (Staging Area). Ты как бы говоришь Гиту: «Подготовь эти файлы, я хочу их сохранить».
        
- **`git commit -m "Твое сообщение"`** (_зафиксировать_)
    
    - **Что делает:** Делает снимок (сохранение) твоего кода намертво. В кавычках обязательно пиши понятный текст (например: `"создал главную страницу"`), чтобы потом вспомнить, что это за сохранение.


📌 **Твой идеальный рабочий план в консоли сейчас выглядит так:**

1. Открыл Git Bash.
    
2. С помощью `cd` дошел до нужной папки.
    
3. Создал файл через `touch`.
    
4. Проверил `git status`.
    
5. Добавил в индекс через `git add .`.
    
6. Сохранил через `git commit -m "сообщение"`.
