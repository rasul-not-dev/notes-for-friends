
### 📌 Что такое ветки (Branches)?

- **Ветка** — это независимая линия разработки. Можно представить её как параллельную вселенную твоего проекта.
    
- Ветки позволяют безопасно экспериментировать, добавлять новые функции или исправлять баги, не ломая основной рабочий код.
    
- Технически ветка в Git — это просто **подвижный указатель** на один из коммитов.
    

### 🌳 Основная ветка (main / master)

- Когда ты инициализируешь репозиторий, Git автоматически создает первую базовую ветку.
    
- Исторически она называлась **`master`**, но сейчас стандартом (особенно на GitHub) стало название **`main`**.
    
- **Золотое правило:** В главной ветке всегда должен находиться только полностью рабочий, готовый и протестированный код. Всю новую работу нужно вести в побочных ветках.
    

### ⚙️ Как работает ветвление под капотом

- В Git есть специальный системный указатель **`HEAD`**. Он всегда смотрит на ту ветку (и тот коммит), в которой ты находишься в данный момент.
    
- Когда ты делаешь новый коммит, указатель твоей текущей ветки автоматически сдвигается вперед вместе с `HEAD`.
    

### 💻 Базовые команды для работы с ветками

**1. Просмотр списка веток**

> `git branch`

- Показывает список всех веток в твоем локальном репозитории.
    
- Текущая ветка (в которой ты находишься) будет отмечена звездочкой `*` и выделена цветом.
    

**2. Создание новой ветки**

> `git branch <имя_ветки>`

- _Пример:_ `git branch feature-payment` (создает ветку для разработки системы оплаты).
    
- **Важно:** Эта команда только _создает_ ветку, но **не** переключает тебя на неё. `HEAD` остается на старом месте.
    

**3. Переключение между ветками**

> `git checkout <имя_ветки>`

- Перемещает указатель `HEAD` на указанную ветку. Файлы в твоей рабочей папке на компьютере мгновенно изменятся на те, что сохранены в выбранной ветке.
    
- _Альтернатива:_ В новых версиях Git добавили более логичную команду `git switch <имя_ветки>`. Работает точно так же.
    

**4. Создание и переключение одной командой (Самая частая практика)**

> `git checkout -b <имя_ветки>`

- Флаг `-b` означает "создать и сразу переключиться".
    
- _Альтернатива:_ `git switch -c <имя_ветки>`.
    

### 💡 Главный вывод урока

Создание веток в Git происходит мгновенно и совершенно не утяжеляет проект (в отличие от копирования папок на компьютере). Поэтому главная привычка разработчика — **создавать новую ветку для любой, даже самой мелкой задачи**.






**ШПАРГАЛКА**

- `git branch` — посмотреть список веток (твоя текущая будет со звездочкой `*`).
    
- `git branch <имя_ветки>` — создать новую ветку, но остаться в старой.
    
- `git checkout <имя_ветки>` (или `git switch <имя_ветки>`) — переключиться на другую ветку.
    
- `git checkout -b <имя_ветки>` (или `git switch -c <имя_ветки>`) — создать ветку и сразу на нее переключиться.
    

**Суть на пальцах:** `main` (или `master`) — это всегда рабочий код. Для любой новой задачи создаешь отдельную ветку. **HEAD** — это просто указатель, который отмечает ветку, в которой ты сейчас находишься. Создание веток в Git происходит мгновенно и не занимает места на диске.
