# Наследование шаблонов в Django: `extends`, `block` и `include`

При создании сайтов многие элементы страниц (шапка, подвал, боковое меню) дублируются. Чтобы соблюдать принцип **DRY** (_Don't Repeat Yourself_ — Не повторяйся), в Django используется механизм **наследования шаблонов**.

## 1. Базовый шаблон (Base Template)

Базовый шаблон содержит общую структуру HTML-страницы (каркас). В тех местах, где у разных страниц должен быть свой уникальный контент, объявляются **блоки** с помощью тега `{% block %}`.

### Создание базового шаблона `base.html`:

HTML

```
<!DOCTYPE html>
<html lang="ru">
<head>
    <meta charset="UTF-8">
    <title>{% block title %}Мой сайт{% endblock %}</title>
</head>
<body>
    <header>
        <!-- Общая шапка сайта -->
        <nav>
            <a href="/">Главная</a> | <a href="/about/">О нас</a>
        </nav>
    </header>

    <main>
        <!-- Сюда дочерние шаблоны будут вставлять свой контент -->
        {% block content %}
        {% endblock %}
    </main>

    <footer>
        <p>&copy; 2026 Все права защищены</p>
    </footer>
</body>
</html>
```

### Настройка путей для общих шаблонов

Если базовый шаблон лежит в корневой папке проекта (например, `templates/base.html`), этот путь нужно зарегистрировать в `settings.py` в секции `TEMPLATES` -> `DIRS`:

Python

```
TEMPLATES = [
    {
        ...
        'DIRS': [BASE_DIR / 'templates'], # Указываем путь к общей папке шаблонов
        ...
    },
]
```

## 2. Дочерние шаблоны и тег `{% extends %}`

Дочерний шаблон "наследует" структуру базового с помощью тега `{% extends %}`. Он переопределяет только те блоки, которые ему необходимы.

### Важные правила:

1. Тег `{% extends "base.html" %}` должен быть **самой первой строчкой** в дочернем шаблоне.
    
2. Весь уникальный HTML-код дочернего шаблона должен находиться строго внутри блоков `{% block %}`. Всё, что написано вне блоков, Django просто проигнорирует.
    

### Пример дочернего шаблона `about.html`:

HTML

```
{% extends "base.html" %}

{% block title %}О нас - Мой сайт{% endblock %}

{% block content %}
    <h1>О нашей компании</h1>
    <p>Мы разрабатываем крутые веб-приложения на Django!</p>
{% endblock %}
```

## 3. Подключение шаблонов с помощью `{% include %}`

Тег `{% include %}` используется для вставки содержимого одного HTML-файла внутрь другого. Это полезно для изоляции мелких, повторяющихся компонентов (например, боковое меню, карточка товара, форма подписки).

### Синтаксис:

HTML

```
{% include "path/to/sidebar.html" %}
```

### Дополнительные возможности передачи контекста:

- **`with`** — передает локальные переменные внутрь подключаемого шаблона:
    
    HTML
    
    ```
    {% include "includes/banner.html" with text="Скидка 50%!" %}
    ```
    
- **`only`** — изолирует контекст, запрещая подключаемому шаблону видеть переменные родительского шаблона (доступны будут только переменные, переданные через `with`):
    
    HTML
    
    ```
    {% include "includes/banner.html" with text="Привет!" only %}
    ```
    

## Пример использования на практике

Организация структуры шаблонов в реальном приложении:

Plaintext

```
templates/
│
├── base.html              # Каркас сайта (шапка, футер)
│
└── includes/
    └── newsletter_form.html # Форма подписки (повторяющийся блок)
```

### Шаблон `base.html`:

HTML

```
...
<body>
    {% block content %}{% endblock %}
    
    <!-- Подключаем форму подписки перед футером на всех страницах -->
    {% include "includes/newsletter_form.html" %}
</body>
...
```


## Иерархия папок проекта Django при:

```
TEMPLATES = [
    {
        ...
        'DIRS': [BASE_DIR / 'templates'], # Указываем путь к общей папке шаблонов
        ...
    },
]
```


```

my_project/                  # Корневой каталог твоего проекта
│
├── manage.py                # Главный скрипт управления Django
│
├── my_project/              # Папка конфигурации проекта (тут лежит settings.py)
│   ├── __init__.py
│   ├── settings.py          # ТУТ прописываем: 'DIRS': [BASE_DIR / 'templates']
│   ├── urls.py
│   └── wsgi.py
│
├── templates/               # ГЛОБАЛЬНАЯ папка шаблонов (для всего проекта)
│   ├── base.html            # Твой каркас (шапка, футер, блоки {% block %})
│   │
│   └── includes/            # Папка для переиспользуемых компонентов
│       └── alert_box.html   # Компонент плашки уведомления ({% include %})
│
└── main/                    # Твое Django-приложение (например, "главное")
    ├── __init__.py
    ├── views.py
    ├── urls.py
    └── templates/           # Локальные шаблоны приложения
        └── main/            # Папка с именем приложения (защита от совпадения имен)
            └── profile.html # Дочерний шаблон ({% extends 'base.html' %})
```

## Шпаргалка

- Наследование: {% extends 'base.html' %} — пишется строго первой строкой в дочернем шаблоне.
- Область для контента: {% block block_name %} ... {% endblock %} — задается в родителе, заполняется в дочернем шаблоне.
- Вставка компонента: {% include 'path/template.html' %} — вставляет один HTML в другой.
- Передача параметров в include: {% include 'tmpl.html' with key="val" %}
- Полная изоляция в include: {% include 'tmpl.html' with key="val" only %}
