
**Главная тема урока:** Как перенаправить пользователя с одного URL-адреса на другой (например, если страница переехала) и как гибко формировать адреса в коде без жесткой привязки к тексту (без хардкода).

## 1. Перенаправления (Redirects) и HTTP-статусы

Перенаправление нужно для того, чтобы автоматически перекинуть пользователя на другой URL-адрес. В веб-разработке для этого используются два основных статус-кода:

- **302 Found (Временное перенаправление):** Поисковые роботы понимают, что страница переехала _временно_, и не удаляют старый адрес из индекса. В Django используется по умолчанию.
    
- **301 Moved Permanently (Постоянное перенаправление):** Сообщает поисковикам, что страница переехала _навсегда_. Это важно для SEO, чтобы передать «вес» старой страницы новому адресу.
    

### Функция `redirect()`

Самый простой и гибкий способ сделать редирект — использовать функцию `redirect` из модуля `django.shortcuts`. Она всегда возвращается через оператор `return`.

- **Пример 302 редиректа на главную страницу (прямой URL):**
    
    Python
    
    ```
    from django.shortcuts import redirect
    
    def archive(request, year):
        if year > 2023:
            return redirect('/') # Временный редирект (302) на главную страницу
    ```
    
- **Пример 301 редиректа (постоянного):** Для этого нужно передать именованный аргумент `permanent=True`.
    
    Python
    
    ```
    return redirect('/', permanent=True) # Постоянный редирект (301)
    ```
    

### Что еще может принимать `redirect()` в качестве аргумента?

Функция очень гибкая. Вместо жестко прописанной строки URL (`'/'`) ей можно передавать:

1. **Функцию представления:** `return redirect(index)` (Django сам поймет, какой URL закреплен за функцией `index`).
    
2. **Имя маршрута:** `return redirect('home')` (если у маршрута задан параметр `name`).
    

## 2. Именование маршрутов (Параметр `name`)

Прописывать конкретные URL-адреса (типа `'/cats/'`) прямо в коде — это **плохая практика (хардкод)**. Если в будущем вы решите изменить адрес с `/cats/` на `/categories/`, вам придется искать и вручную менять эту строку по всему проекту.

**Решение:** Давать маршрутам уникальные имена с помощью параметра `name` в файле `urls.py`.

Python

```
# urls.py
path('', views.index, name='home'),                           # Имя маршрута: home
path('cats/<slug:cat_slug>/', views.categories, name='cats'), # Имя маршрута: cats
```

Теперь в коде `views.py` можно делать перенаправление по имени:

Python

```
# Перенаправление на маршрут с динамическим параметром (slug)
# Django сам подставит 'music' в URL-адрес и сделает переход на /cats/music/
return redirect('cats', cat_slug='music') 
``` 

## 3. Функция `reverse()` (Обратное разрешение URL)

Иногда нам нужно не сразу перенаправить пользователя, а просто **получить (вычислить) строковый URL-адрес** по имени маршрута, чтобы использовать его позже в логике программы. Для этого служит функция `reverse()`.

- **Откуда импортировать:** `from django.urls import reverse`
    
- **Как работает:** Она принимает имя маршрута и возвращает готовую строку адреса.
    

### Передача аргументов в `reverse()`

Если у маршрута есть динамические параметры (например, `<int:...>` или `<slug:...>`), их нужно передать в `reverse` через специальный аргумент **`args`** в виде коллекции (списка или кортежа).

Python

```
from django.urls import reverse
from django.shortcuts import redirect

def archive(request, year):
    if year > 2023:
        # Вычисляем URL для маршрута с именем 'cats' и передаем туда параметр 'sport'
        url_address = reverse('cats', args=['sport']) # Вернет строку: '/cats/sport/'
        
        # Передаем вычисленный адрес в редирект
        return redirect(url_address) 
``` 

## 4. Классы `HttpResponseRedirect` и `HttpResponsePermanentRedirect`

Помимо функции `redirect()`, Django предоставляет два низкоуровневых класса для перенаправлений:

- `HttpResponseRedirect(url)` — делает 302 редирект.
    
- `HttpResponsePermanentRedirect(url)` — делает 301 редирект.
    

Они импортируются из `django.http` и работают аналогично:

Python

```
from django.http import HttpResponseRedirect
from django.urls import reverse

def my_view(request):
    url = reverse('home')
    return HttpResponseRedirect(url) # Перенаправление на главную страницу (302)
```

### Что использовать на практике?

**Функция `redirect()` предпочтительнее**, так как она более гибкая. В отличие от классов, которые принимают _только_ готовый строковый URL, функция `redirect()` может автоматически «на лету» определять адреса по именам маршрутов и функциям представлений.
