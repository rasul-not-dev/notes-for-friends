# Шаблонный тег `{% url %}` в Django

**Проблема хардкода:** Если прописать ссылки в HTML вручную (например, `<a href="/post/5/">`), а потом изменить структуру URL в `urls.py`, проект сломается. Придется вручную искать и переписывать ссылки во всех шаблонах.

**Решение:** Шаблонный тег `{% url %}` генерирует пути динамически на основе **имени маршрута** (аргумент `name` в `urls.py`). Он работает по аналогии с функцией `reverse()` в Python-коде Django.

## 1. Синтаксис тега `{% url %}`

### Без параметров (простой маршрут)

Если у маршрута нет динамических частей:

HTML

```
<a href="{% url 'имя_маршрута' %}">Текст ссылки</a>
```

### С параметрами (динамический маршрут)

Если маршрут принимает переменные (например, `id` или `slug`), они передаются через пробел после имени маршрута:

HTML

```
<a href="{% url 'имя_маршрута' параметр1 параметр2 %}">Читать далее</a>
```

_В качестве параметров могут выступать свойства объектов из контекста шаблона (например, `p.id`)._

## 2. Динамическое построение меню

Вместо верстки каждого пункта меню вручную, структуру меню можно передать из `views.py` в виде списка словарей, а затем отрендерить его в цикле.

### Структура данных в Python (`views.py`):

Python

```
menu = [
    {"title": "О сайте", "url_name": "about"},
    {"title": "Добавить статью", "url_name": "add_page"},
    {"title": "Обратная связь", "url_name": "contact"},
    {"title": "Войти", "url_name": "login"},
]
```

### Рендеринг в HTML-шаблоне (`index.html`):

Если имя маршрута хранится в переменной цикла, кавычки в теге `{% url %}` **не ставятся**:

HTML

```
<ul>
    {% for m in menu %}
        <li>
            <a href="{% url m.url_name %}">{{ m.title }}</a>
        </li>
    {% endfor %}
</ul>
```

## Пример использования на практике

Давай настроим связку маршрутов, представления и шаблона с использованием тега `{% url %}`.

### 1. Определение маршрутов (`urls.py`):

Python

```
from django.urls import path
from . import views

urlpatterns = [
    path('', views.index, name='home'),
    path('post/<int:post_id>/', views.show_post, name='post'),
    path('about/', views.about, name='about'),
]
```

### 2. Шаблон списка статей с динамическими ссылками (`index.html`):

HTML

```
<nav>
    <a href="{% url 'home' %}">Главная</a> | 
    <a href="{% url 'about' %}">О проекте</a>
</nav>

<hr>

<h2>Список статей</h2>
<ul>
    {% for p in posts %}
        <li>
            <h3>{{ p.title }}</h3>
            <p>{{ p.content }}</p>
            {# Передаем p.id как параметр для маршрута 'post' #}
            <a href="{% url 'post' p.id %}">Читать пост полностью</a>
        </li>
    {% endfor %}
</ul>
```

## Шпаргалка

- Статический URL: {% url 'имя_маршрута' %} (имя обязательно в кавычках).
- Динамический URL с ID: {% url 'имя_маршрута' p.id %} (параметры пишутся через пробел).
- URL из переменной: {% url variable_name %} (без кавычек, если имя маршрута лежит в переменной).
- На чемпионате: ВСЕГДА используй {% url %} вместо ручного прописывания href="/path/".
