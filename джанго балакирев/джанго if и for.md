# Теги шаблонов Django: `for` и `if`

В Django шаблонизатор (Django Template Language — DTL) позволяет не просто выводить переменные, но и добавлять базовую логику прямо в HTML-код с помощью **тегов**.

## 1. Синтаксис тегов в Django

В отличие от переменных, которые выводятся в двойных фигурных скобках `{{ variable }}`, теги Django записываются с использованием фигурных скобок и знаков процента:

HTML

```
{% имя_тега параметры %}
```

Большинство управляющих тегов (таких как `for` и `if`) являются парными и требуют обязательного закрывающегося тега.

## 2. Цикл `{% for %}`

Используется для перебора коллекций (списков, кортежей, словарей), переданных из Python-представления (views) в контекст шаблона.

### Синтаксис:

HTML

```
{% for item in list %}
    <!-- Тело цикла -->
{% endfor %}
```

_Все, что находится между `for` и `endfor`, многократно дублируется для каждого элемента коллекции._

### Объект `forloop`

Внутри любого цикла `{% for %}` автоматически доступна специальная переменная `forloop`, которая содержит полезные свойства о текущей итерации цикла:

|**Свойство**|**Описание**|
|---|---|
|`forloop.counter`|Счетчик итераций, начиная с **1**.|
|`forloop.counter0`|Счетчик итераций, начиная с **0**.|
|`forloop.revcounter`|Обратный счетчик (до 1).|
|`forloop.revcounter0`|Обратный счетчик (до 0).|
|`forloop.first`|Возвращает `True`, если это **первая** итерация цикла.|
|`forloop.last`|Возвращает `True`, если это **последняя** итерация цикла.|
|`forloop.parentloop`|Ссылка на родительский цикл (при вложенных циклах).|

## 3. Условный оператор `{% if %}`

Позволяет гибко управлять тем, какой HTML-код будет сгенерирован сервером и отправлен пользователю, в зависимости от выполнения условий.

### Синтаксис:

HTML

```
{% if condition %}
    <!-- Код, если условие истинно -->
{% elif other_condition %}
    <!-- Код, если истинно другое условие -->
{% else %}
    <!-- Код во всех остальных случаях -->
{% endif %}
```

_Важно: как и в случае с циклом, обязательно нужно закрывать условие тегом `{% endif %}`._

В условиях можно использовать логические операторы: `and`, `or`, `not`, а также операторы сравнения.

## Пример использования на практике

Давай представим, что мы передаем из `views.py` список постов (словарей) в шаблон.

### Python-представление (`views.py`):

Python

```
from django.shortcuts import render

def index(request):
    data_db = [
        {"id": 1, "title": "Введение в Django", "content": "Текст статьи...", "is_published": True},
        {"id": 2, "title": "Секреты CSS-гридов", "content": "Текст статьи...", "is_published": False},
        {"id": 3, "title": "Работа с Git", "content": "Текст статьи...", "is_published": True},
    ]
    context = {
        'posts': data_db,
        'title': 'Главная страница'
    }
    return render(request, 'index.html', context=context)
```

### HTML-шаблон (`index.html`):

HTML

```
<h1>{{ title }}</h1>

<ul>
    {% for p in posts %}
        {# Отображаем только опубликованные статьи #}
        {% if p.is_published %}
            <li>
                <h2>{{ p.title }}</h2>
                <p>{{ p.content }}</p>
                
                {# Выводим разделитель HR везде, кроме последней статьи в списке #}
                {% if not forloop.last %}
                    <hr>
                {% endif %}
            </li>
        {% endif %}
    {% endfor %}
</ul>
```

## Шпаргалка

- Переменные: {{ variable }} (просто вывод данных)
- Теги Django: {% tag_name %} (логика: циклы, условия)
- Цикл: {% for item in items %} ... {% endfor %}
- Исключить последний элемент: {% if not forloop.last %} ... {% endif %}
- Проверка условий: {% if condition %} ... {% elif cond2 %} ... {% else %} ... {% endif %}
- Логическое отрицание в шаблоне: not (например: if not p.is_published)
