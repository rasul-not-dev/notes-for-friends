
## 1. Встроенные конвертеры (int и slug)

Когда мы хотим, чтобы URL-адрес менялся динамически (например, выводить разные категории или статьи), мы используем угловые скобки `<тип_конвертера:имя_переменной>`.

### Как работает `int`

Python

```
# urls.py
path('cats/<int:cat_id>/', views.categories)
```

- **Суть:** Если в URL на месте `<int:cat_id>` стоит **целое число**, Django вырезает его, автоматически создает переменную `cat_id` и передает её в функцию во `views.py`.
    
- В файле `views.py` функция **обязательно** должна принимать этот параметр с тем же именем:
    
    Python
    
    ```
    def categories(request, cat_id):
        return HttpResponse(f"<h1>ID категории: {cat_id}</h1>")
    ```
    

### Что такое слаг (`slug`)?

**Слаг (slug)** — это часть URL-адреса, написанная понятными человеческими словами вместо ID-номеров (очень полезно для поисковиков и красоты).

- **С ID:** `[mysite.com/cats/3/](https://mysite.com/cats/3/)`
    
- **Со слагом:** `[mysite.com/cats/music-for-studying/](https://mysite.com/cats/music-for-studying/)`
    

Конвертер `slug` — это встроенный фильтр, который пропускает в URL **только**:

1. Латинские буквы (`a-z`, `A-Z`)
    
2. Цифры (`0-9`)
    
3. Дефис (`-`)
    
4. Нижнее подчеркивание (`_`)
    

Если вбить пробел или русский символ — Django выдаст ошибку 404.

### Важнейшее правило: Порядок маршрутов

Django проверяет список маршрутов в `urls.py` **сверху вниз**. Как только находится первое совпадение, он останавливается.

**⚠️ Ошибка смены мест:**

Python

```
path('cats/<slug:cat_slug>/', views.categories_by_slug), # Неправильно ставить ПЕРВЫМ
path('cats/<int:cat_id>/', views.categories),
```

Обычное число (например, `3`) — это технически тоже частный случай слага. Если пользователь введет `cats/3/`, Django зацепится за верхний маршрут `slug`, посчитает его совпавшим и вызовет `categories_by_slug`. До нижнего `int` код **никогда не дойдет**.

> **Правило:** Всегда пиши сначала **более специфичные/узкие** конвертеры (например, `int`), а ниже — **более общие** (например, `slug` или `str`).

## 2. Функция re_path (Регулярные выражения)

Если встроенных фильтров не хватает (например, нам нужно поймать в URL **строго 4 цифры** для года: `archive/2026/`), используется функция `re_path()`.

Python

```
from django.urls import re_path
from . import views

urlpatterns = [
    re_path(r'^archive/(?P<year>[0-9]{4})/$', views.archive),
]
```

- `(?P<year>[0-9]{4})` — создает переменную `year`, в которой должно быть **ровно 4 цифры** (от 0 до 9).
    
- Минус подхода: код становится «грязным» и трудночитаемым из-за обилия спецсимволов.
    

## 3. Пользовательские конвертеры (Подробный разбор)

Чтобы не писать страшные регулярные выражения прямо в `urls.py`, мы можем создать **свой собственный конвертер** (например, с именем `year4`), настроить его логику один раз, а потом использовать так же легко, как `int` или `slug`.

Реализуется это в 3 шага.

### Шаг 1. Создаем класс конвертера

Внутри твоего приложения создается отдельный файл `converters.py`. В него пишется класс:

Python

```
class FourDigitYearConverter:
    # 1. Правило отбора (Регулярное выражение)
    regex = '[0-9]{4}'  # Означает: строго 4 цифры подряд от 0 до 9

    # 2. Превращение ТЕКСТА из URL в данные для Питона
    def to_python(self, value):
        # Изначально из URL к нам прилетает строка "2026" (в кавычках).
        # Этот метод автоматически переводит её в целое число (int) 2026.
        # И именно число прилетит во views.py.
        return int(value)

    # 3. Превращение данных Питона обратно в ТЕКСТ для URL
    def to_url(self, value):
        # Нужно для автоматической генерации ссылок в шаблонах Django.
        # Берет число и возвращает его в виде строки.
        return '%04d' % value
```

### Шаг 2. Регистрируем конвертер в `urls.py`

Сам по себе класс работать не будет, Django нужно с ним познакомить. Открываем `urls.py`:

Python

```
from django.urls import path, register_converter
from . import views
from . import converters  # Импортируем наш файл с классом

# Регистрируем: связываем наш класс с придуманным коротким именем 'year4'
register_converter(converters.FourDigitYearConverter, 'year4')
```

### Шаг 3. Используем в списке маршрутов

Теперь `'year4'` — полноценный конвертер. Пишем красивый и понятный маршрут:

Python

```
urlpatterns = [
    # Вместо страшных регулярок или глупого <int:year>
    path('archive/<year4:year>/', views.archive),
]
```

### 🔁 Как вся эта цепочка работает на практике (пошагово):

1. Пользователь переходит по адресу: `запрос -> /archive/2026/`.
    
2. Django заходит в `urls.py`, видит фильтр `<year4:year>`.
    
3. Заглядывает в класс конвертера и проверяет строку `"2026"` по правилу `regex = '[0-9]{4}'`. Цифр ровно четыре? **Да.** Проверка пройдена. Если бы цифр было 3 или 5, Django сразу бы выдал ошибку 404.
    
4. Django берет текстовую строку `"2026"` и прогоняет через метод `to_python()`. Метод превращает её в математическое число `2026`.
    
5. Django сам вызывает функцию `archive` из `views.py` и передает туда готовое число: `views.archive(request, year=2026)`.
