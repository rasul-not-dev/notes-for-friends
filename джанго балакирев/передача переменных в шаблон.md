# Передача данных (переменных) в шаблоны Django

Чтобы сделать HTML-страницы динамическими, Django позволяет передавать переменные из Python-кода (функций представления) напрямую в шаблонизатор.

## 1. Синтаксис вывода переменных в HTML

Для вывода значения переменной в шаблоне используются **двойные фигурные скобки** `{{ ... }}`. Внутри скобок указывается имя ключа, переданного из представления.

HTML

```
<h1>{{ title }}</h1>
<p>Добро пожаловать на {{ title }}!</p>
```

_Совет:_ Пробелы внутри скобок ставить не обязательно, но крайне рекомендуется для читаемости кода.

## 2. Передача контекста через словарь

Данные передаются в функцию `render()` в виде обычного Python-словаря (который принято называть `context` или `data`). Ключи этого словаря становятся переменными внутри шаблона.

Python

```
# views.py
from django.shortcuts import render

def index(request):
    data = {
        'title': 'Главная страница',
        'menu': ['О сайте', 'Добавить статью', 'Войти']
    }
    # Способ 1: передача словаря третьим позиционным аргументом
    return render(request, 'women/index.html', data)

    # Способ 2: явная передача через именованный параметр context (предпочтительно для ясности)
    # return render(request, 'women/index.html', context=data)
```

### Важное поведение Django:

Если переменная прописана в HTML-шаблоне (например, `{{ menu }}`), но не была передана в словаре контекста из `views.py`, Django **не выбросит ошибку**, а просто проигнорирует её, выведя пустую строку.

## 3. Передача различных типов данных и синтаксис "Точки" (`.`)

В шаблоны Django можно передавать абсолютно любые типы данных Python: числа, строки, списки, множества, словари и даже объекты пользовательских классов.

**Важно:** Шаблонизатор Django — это не чистый Python. У него свой синтаксис. Классический синтаксис Python для работы с коллекциями и объектами здесь выдаст ошибку `TemplateSyntaxError`. Вместо этого используется **универсальный синтаксис точки (`.`)**.

### Таблица синтаксиса: Python vs Шаблонизатор Django

|**Тип данных**|**Как пишем в Python**|**Как пишем в шаблоне Django**|
|---|---|---|
|**Словарь**|`my_dict['key']`|`{{ my_dict.key }}`|
|**Объект класса**|`my_obj.attribute`|`{{ my_obj.attribute }}`|
|**Список (по индексу)**|`my_list[0]`|`{{ my_list.0 }}` _(обращение по индексу через точку)_|

### Ограничения шаблонизатора:

Внутри одних фигурных скобок `{{ ... }}` нельзя выводить несколько переменных сразу или писать сложные выражения (например, `{{ var1 var2 }}`). Для каждой переменной должны быть свои скобки: `{{ var1 }} {{ var2 }}`.

## Пример использования на практике

Передача сложного набора данных в шаблон и их выборочный вывод.

Python

```
# views.py
from django.shortcuts import render

class Student:
    def __init__(self, name, age):
        self.name = name
        self.age = age

def student_profile(request):
    context = {
        'title': 'Профиль студента',
        'score': 4.85,
        'skills': ['Python', 'Django', 'Git'],
        'achievements': {'olympiad': '1st place', 'hackathon': 'Finalist'},
        'student_info': Student('Расул', 16)
    }
    return render(request, 'blog/profile.html', context=context)
```

Шаблон `blog/templates/blog/profile.html`:

HTML

```
<!DOCTYPE html>
<html lang="ru">
<head>
    <meta charset="UTF-8">
    <title>{{ title }}</title>
</head>
<body>
    <h1>Ученик: {{ student_info.name }} (Возраст: {{ student_info.age }})</h1>
    <p>Средний балл: {{ score }}</p>
    
    <!-- Доступ по индексу к списку -->
    <p>Основной навык: {{ skills.0 }}</p> 
    
    <!-- Доступ к ключу словаря -->
    <p>Олимпиады: {{ achievements.olympiad }}</p> 
</body>
</html>
```

## Шпаргалка

- Вывод переменной: `{{ variable_name }}`.
    
- Обращение к ключам словарей или свойствам объектов — только через точку: `{{ dict.key }}` или `{{ object.property }}`.
    
- Квадратные скобки `[]` для индексов списков использовать **нельзя**. Вместо этого пишем `{{ list.0 }}`.
    
- Если переменной нет в контексте, Django молча пропустит её (ошибки не будет).
