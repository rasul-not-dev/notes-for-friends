# Стандартные шаблонные фильтры в Django

**Шаблонные фильтры** (template filters) — это специальные функции, которые применяются к переменным в шаблоне для изменения их визуального представления.

## 1. Синтаксис применения фильтров

Фильтры записываются внутри двойных фигурных скобок `{{ ... }}`. Для применения используется вертикальная черта `|` (pipe).

HTML

```
{{ имя_переменной|имя_фильтра }}
```

Если фильтр принимает аргументы, они передаются после двоеточия `:` без пробелов:

HTML

```
{{ имя_переменной|имя_фильтра:аргумент }}
```

_Важно:_ Аргументы обычно оборачиваются в кавычки, чтобы Django корректно считывал и строки, и числа.

### Цепочки фильтров (Chaining)

Фильтры можно объединять в цепочки, выполняя их последовательно слева направо:

HTML

```
{{ title|cut:"?"|upper }}
```

_(Сначала из переменной `title` удалятся знаки вопроса, а затем вся строка будет приведена к верхнему регистру)_.

## 2. Основные встроенные фильтры

Ниже представлены самые ходовые фильтры, которые пригодятся на практике.

### Работа с числами

- **`add`** — прибавляет число к переменной.
    
    HTML
    
    ```
    {{ score|add:10 }} <!-- К score прибавится 10 -->
    ```
    
    _Особенность:_ Если применить к числу с плавающей точкой (`float`), Django сначала преобразует его в целое число (отбросив дробную часть), а затем сложит.
    
- **`divisibleby`** — проверяет, делится ли число нацело на указанный делитель. Возвращает `True` или `False`.
    
    HTML
    
    ```
    {{ page_num|divisibleby:2 }} <!-- Проверка на четность -->
    ```
    

### Работа со строками

- **`capfirst`** — делает первую букву строки заглавной.
    
    HTML
    
    ```
    {{ "привет"|capfirst }} <!-- Результат: "Привет" -->
    ```
    
- **`upper` / `lower`** — переводит всю строку в верхний или нижний регистр соответственно.
    
- **`cut`** — удаляет из строки указанный фрагмент или символ.
    
    HTML
    
    ```
    {{ "hello world"|cut:" " }} <!-- Результат: "helloworld" (удалит все пробелы) -->
    ```
    
- **`slugify`** — преобразует строку в латиницу (если применимо), переводит в нижний регистр и заменяет пробелы дефисами (создает "слаг" для URL).
    

### Работа со списками и коллекциями

- **`first` / `last`** — выводит первый или последний элемент списка/коллекции.
    
    HTML
    
    ```
    {{ menu|first }} <!-- Выведет первый пункт меню -->
    ```
    
- **`length`** — возвращает длину (количество элементов) списка или строки.
    
    HTML
    
    ```
    {{ users|length }} <!-- Выведет количество пользователей -->
    ```
    
- **`join`** — объединяет элементы списка в одну строку через указанный разделитель.
    
    HTML
    
    ```
    {{ tags|join:", " }} <!-- Результат: "django, python, backend" -->
    ```
    

### Значения по умолчанию

- **`default`** — если переменная возвращает ложное значение (`False`, `None`, пустую строку, пустой список), фильтр выведет указанное дефолтное значение.
    
    HTML
    
    ```
    {{ avatar|default:"/static/images/default.png" }}
    ```
    

## 3. Использование фильтров прямо в Python-коде

Иногда логику фильтрации хочется применить еще на этапе формирования данных в `views.py`. Все встроенные фильтры Django можно импортировать как обычные функции из модуля `django.template.defaultfilters`.

Python

```
# views.py
from django.shortcuts import render
from django.template.defaultfilters import slugify # Импорт фильтра

def post_view(request):
    raw_title = "My New Post"
    # Применяем фильтр slugify прямо в Python
    post_slug = slugify(raw_title) # вернет "my-new-post"
    
    return render(request, 'blog/post.html', {'slug': post_slug})
```

## Пример использования на практике

Сценарий: вывод тегов статьи красивой строкой, форматирование заголовка и подмена отсутствующего описания.

Python

```
# views.py
from django.shortcuts import render

def article_detail(request):
    context = {
        'title': 'первый шаг в django разработки',
        'content': '',  # пустое содержимое статьи
        'tags': ['web', 'django', 'backend', 'learning'],
        'views': 15.7,   # дробное число просмотров
    }
    return render(request, 'blog/article.html', context=context)
```

Шаблон `blog/templates/blog/article.html`:

HTML

```
<!DOCTYPE html>
<html lang="ru">
<head>
    <meta charset="UTF-8">
    <title>{{ title|capfirst }}</title> <!-- "Первый шаг в django разработки" -->
</head>
<body>
    <h1>{{ title|upper }}</h1> <!-- "ПЕРВЫЙ ШАГ В DJANGO РАЗРАБОТКИ" -->
    
    <!-- Подменяем пустое содержимое дефолтным текстом -->
    <p>{{ content|default:"Извините, текст статьи еще не написан." }}</p>
    
    <!-- Выводим теги через запятую -->
    <p>Теги статьи: {{ tags|join:", " }}</p> 
    
    <!-- Складываем округленные просмотры с бонусом -->
    <p>Всего просмотров с бонусом: {{ views|add:5 }}</p> <!-- 15 + 5 = 20 -->
</body>
</html>
```

## Шпаргалка

- **Синтаксис:** `{{ variable|filter:arg }}` (без пробелов возле двоеточия).
    
- **Цепочки:** `{{ var|filter1|filter2 }}`.
    
- **Популярные строковые фильтры:** `capfirst` (первая буква заглавная), `upper`/`lower` (регистр), `cut` (удаление символа).
    
- **Дефолтные значения:** `{{ text|default:"Текст отсутствует" }}`.
    
- **Списки:** `first` (первый элемент), `last` (последний элемент), `length` (длина), `join` (объединение в строку).
