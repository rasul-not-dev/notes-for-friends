# Подключение статических файлов в Django

В Django к статическим файлам относятся CSS-стили, JavaScript-скрипты, шрифты и изображения. Процесс работы с ними делится на два этапа: разработка на локальном компьютере и деплой на реальный сервер (продакшен).

## 1. Локальная разработка: Статика в приложениях (Режим по умолчанию)

Когда ты только создаешь проект и верстаешь страницы (`DEBUG = True`), Django автоматически ищет статические файлы внутри каждого твоего приложения.

### Шаг 1: Создание структуры папок

Чтобы избежать путаницы между файлами разных приложений, внутри папки твоего приложения всегда создается вложенная структура с именем этого приложения:

Plaintext

```
women/ (твое приложение)
└── static/
    └── women/          <-- папка-изолятор для избежания конфликтов имён
        ├── css/
        │   └── style.css
        ├── js/
        │   └── main.js
        └── images/
            └── photo.jpg
```

### Шаг 2: Подключение статики в HTML-шаблоне

Чтобы Django понял, откуда брать файлы, в шаблоне нужно сделать две вещи:

1. В самом верху файла написать `{% load static %}` (это загружает специальный тег).
    
2. Заменить обычные пути на конструкцию `{% static 'путь_к_файлу' %}`.
    

**Пример в шаблоне (`base.html`):**

HTML

```
{% load static %}
<!DOCTYPE html>
<html lang="ru">
<head>
    <meta charset="UTF-8">
    <title>Главная</title>
    <!-- Подключаем CSS -->
    <link rel="stylesheet" href="{% static 'women/css/style.css' %}">
</head>
<body>
    <!-- Подключаем картинку -->
    <img src="{% static 'women/images/photo.jpg' %}" alt="Фото">

    <!-- Подключаем JS -->
    <script src="{% static 'women/js/main.js' %}"></script>
</body>
</html>
```

## 2. Продвинутая настройка: Работа с переменными в settings.py

Когда базовое подключение работает, можно настроить проект более гибко с помощью специальных переменных в `settings.py`.

### `STATIC_URL`

- **Что делает:** Задает префикс в адресной строке браузера для статики (по умолчанию `static/`).
    
- **Как работает:** Когда в шаблоне ты пишешь `{% static 'women/css/style.css' %}`, Django превратит это в URL `/static/women/css/style.css`.
    

### `STATICFILES_DIRS` (Общая статика проекта)

Если у тебя есть файлы, которые относятся не к конкретному приложению, а ко всему сайту вообще (например, общие стили для админки и фронтенда), их хранят в корне проекта.

1. Создай папку `static_global` в корне проекта (рядом с `manage.py`).
    
2. Пропиши путь к ней в `settings.py`:
    

Python

```
STATICFILES_DIRS = [
    BASE_DIR / "static_global",
]
```

## 3. Подготовка к продакшену (Режим эксплуатации)

Когда сайт готов и ты выключаешь режим отладки (`DEBUG = False`), Django **полностью перестает** раздавать статику самостоятельно ради безопасности и быстродействия. Этим должен заниматься веб-сервер (например, Nginx).

Чтобы отдать веб-серверу все файлы одной кучей, используется механизм сборки статики.

### Шаг 1: Настройка `STATIC_ROOT`

В `settings.py` указывается папка, куда Django скопирует абсолютно всю статику из всех приложений и папки `STATICFILES_DIRS`:

Python

```
STATIC_ROOT = BASE_DIR / "staticfiles"
```

### Шаг 2: Сбор файлов

Выполни команду в терминале:

Bash

```
python manage.py collectstatic
```

Django создаст папку `staticfiles` в корне проекта и скопирует туда всё до единого файла. Именно эту папку ты потом укажешь в настройках Nginx на сервере.

### Локальное тестирование продакшена

Если нужно проверить работу сайта с `DEBUG = False` на своем компьютере, обычный запуск сервера сбросит стили. Запускай проект специальной командой:

Bash

```
python manage.py runserver --insecure
```

## 4. Бонус: Полезные фильтры шаблонизатора из урока

Инструменты для форматирования текста прямо в HTML-шаблонах:

|**Фильтр / Тег**|**Зачем нужен**|**Пример**|
|---|---|---|
|`truncatewords:N`|Обрезает текст до N слов и ставит многоточие. Удобно для анонсов статей.|`{{ post.content\|truncatewords:15 }}`|
|`linebreaks`|Сохраняет переносы строк (абзацы) из базы данных в HTML-теги.|`{{ post.content\|linebreaks }}`|
|`{% autoescape off %}`|Позволяет выводить HTML-теги из переменной как рабочий код, а не как текст.|`{% autoescape off %}{{ post.title }}{% endautoescape %}`|

## Пример использования на практике

Готовый блок настроек статики в `settings.py` для реального проекта:

Python

```
from pathlib import Path

BASE_DIR = Path(__file__).resolve().parent.parent

# В разработке — True, на сервере — False
DEBUG = True 

INSTALLED_APPS = [
    # ...
    'django.contrib.staticfiles', # Обязательно для работы статики!
    'women.apps.WomenConfig',
]

# 1. URL-префикс для браузера
STATIC_URL = 'static/'

# 2. Где искать общую статику (не привязанную к приложениям)
STATICFILES_DIRS = [
    BASE_DIR / "project_static", 
]

# 3. Куда собирать файлы для продакшена
STATIC_ROOT = BASE_DIR / 'staticfiles' 
``` 




```
myproject/                  <-- Корневая директория твоего проекта
├── manage.py
│
├── myproject/              <-- Папка конфигурации проекта
│   ├── __init__.py
│   ├── settings.py         <-- Тут живут STATIC_URL, STATICFILES_DIRS и STATIC_ROOT
│   ├── urls.py
│   └── wsgi.py
│
├── women/                  <-- Твое Django-приложение
│   ├── migrations/
│   ├── templates/
│   │   └── women/
│   │       └── index.html
│   ├── static/             <-- Локальная статика приложения
│   │   └── women/          <-- Защита от конфликта имен (пространство имен)
│   │       ├── css/
│   │       │   └── style.css
│   │       └── images/
│   │           └── hero.jpg
│   ├── apps.py
│   ├── models.py
│   └── views.py
│
├── global_static/          <-- Глобальная статика (прописана в STATICFILES_DIRS)
│   └── css/
│       └── global.css      <-- Общие стили для всего сайта (например, твоя домашка)
│
└── staticfiles/            <-- Сюда Django САМ соберет файлы после collectstatic (STATIC_ROOT)
    ├── global_static/      <-- Скопировано из глобальной папки
    └── women/              <-- Скопировано из приложения "women"
```


## Шпаргалка

1. Положи стили в: \[приложение]/static/\[приложение]/css/style.css
2. В шаблоне:
   - Сверху: {% load static %}
   - В тегах: \<link rel="stylesheet" href="{% static 'app/css/style.css' %}">
1. Общая папка в корне: укажи её в STATICFILES_DIRS в settings.py
2. Для сервера (DEBUG=False):
   - Пропиши STATIC_ROOT = BASE_DIR / 'staticfiles'
   - Запусти сборку: python manage.py collectstatic
   - Тест локально без DEBUG: python manage.py runserver --insecure
